Learnings

I build on the knowledge I acquired in Fundamentals of Systems Administration with the course Managing Enterprise Servers. The same Ubuntu Linux was primarily used here, but Ansible playbooks Ire utilized in place of bash to enable efficient command execution in a single run. SSH and Git application within the Ubuntu system, along with connecting this to other hosts, is another topic I used that is a new concept. A new operating system called CentOS was also released; it shares some similarities and differences with Ubuntu, which can make using CentOS confusing and challenging at times. Additionally, I gained additional knowledge about the uses and capabilities of Ansible for managing various aspects of servers and hosts, including file management, installation and configuration of various programs and services, and more, all of which contribute to better server management. Containerization using Docker was the final topic covered. The concept is understood, but I'm not sure how this one works exactly. Instead of requiring a separate server, it functions as a standalone image file container that the user can launch to create various profiles appropriate for various tasks and apps. This increases containerization's efficiency to a great degree. Similarly, the final sections of the activities involve installing different services that facilitate server and host management.

Reflection

What I studied in Fundamentals of Systems Administration is expanded upon in the course Managing Enterprise Servers. Here, I mostly used the same Ubuntu Linux, but I used Ansible with the playbooks in place of bash to run the commands efficiently in a single run. Using SSH and Git within the Ubuntu system and connecting it to other hosts is another topic I covered that is a novel idea. In addition, a new operating system called CentOS was released. It shares some similarities and differences with Ubuntu, which can occasionally make using CentOS confusing and challenging.I also gained more knowledge about the uses and capabilities of Ansible for handling various aspects of servers and hosts, including file management, installation, and configuration of various programs and services that aid in more effective server management. Docker-assisted containerization was the final topic covered. I'm not quite sure how this one works, but the idea is clear. This increases containerization's efficiency to a great degree. The final stages of the activity are comparable to installing different services that make managing servers and hosts easier.
